# AIED-DuoGrow-SaaS

一个面向中年人、老年人和带娃家长的英语学习小工具。它把日常场景转成可以直接开口的短句，适合亲子英语、手机端复听、轻量复习和每天留一张记忆卡片。

灵感来源于 `2026.5.10` 送给母亲的母亲节礼物。项目定位为 MIT 开源项目，希望让一个普通家庭也能用很低的成本，把 AI 变成可保存、可复听、可慢慢练习的学习陪伴。

GitHub 仓库：<https://github.com/connectedGraph/AIED-DuoGrow-SaaS>

## 项目形态

- 单 HTML 文件版本：当前 `v1+.html` 可以直接打开，也适合静态部署。
- 拆分代码版本：用于后续工程化维护、主题切换和多版本同步。
- 静态 LLM 调用：前端直接调用 OpenAI-compatible API，不需要自建后端。
- BYOK：用户自带文本模型 API Key 和语音 API Secret。
- 本地存储：学习数据默认保存在浏览器 `localStorage`。

## 功能

- 输入生活场景，生成 5 句或 10 句亲子英语表达。
- 每句包含英文、中文解释和适合使用的场景。
- 保存整组内容到仓库，后续可复看、复听和删除。
- 仓库支持单句播放、整组播放、全仓库循环听和后台听仓库。
- 复习页会从保存过的表达里抽题，记录答对、答错和熟练度。
- 每日一记卡片会从仓库随机抽取内容，生成适合手机保存的图片。
- 保存图支持长按存入相册，方便给家人复看。
- 豆包一对一训练会复制魔法指令，引导豆包按仓库句子做中译英小任务。
- 音频会缓存到本地，重复播放时优先读取缓存，减少重复请求。

## 接口与密码格式

文本生成使用 OpenAI-compatible API。当前 `baseUrl` 和 `model` 写在 HTML 常量中，页面不提供自由配置入口，方便给非技术用户直接使用。

语音使用外部 TTS API，而不是浏览器内置语音、系统朗读或其它免费方案。主要原因是移动端和微信 WebView 的浏览器语音能力不稳定，音色、发音质量和可用性都不容易保证；外部语音 API 更适合做可复听、可缓存的亲子英语体验。

右上角密码框输入的是一整串组合密码，格式类似：

```text
sk-xxxx-yyyy
```

解析规则：

- 前半部分 `sk-xxxx` 会作为 OpenAI-compatible API Key。
- 最后一段 `yyyy` 会作为语音 API Secret。
- 如果 API Key 本身包含更多 `-`，仍然以最后一段作为语音 Secret，前面部分重新拼回文本模型 API Key。
- README 不提供真实 API Key 或真实 Secret，请自行从对应服务商后台获取。

## 本地数据

应用使用 `localStorage` 保存：

- 当前课程
- 仓库主题
- 复习表达
- 每日一记抽取结果
- 生成缓存
- 音频缓存
- 组合密码

更换浏览器、清理浏览器数据或换设备后，本地数据可能不可见。这个版本默认不上传学习记录。

## 部署

当前版本是单文件前端应用，直接打开 `v1+.html` 即可使用，也可以上传到任意静态网页托管服务。

如果需要探索国内静态分发，可以参考 `web2html.com` 这类静态部署/网页封装方式，作为一种免人工审核的静态部署思路。实际使用时请以服务商规则和当地合规要求为准。

## Roadmap

- 同步提供更多主题版本，例如古诗词、中老年英语、亲子阅读和生活照护表达。
- 使用 DDD 思路拆分领域配置，让主题、提示词、词库和 UI 文案可以一键切换。
- 持续维护单 HTML 版本和拆分代码版本。
- 未来可能提供一套 `html2miniprogram` 规则器：填写主题、提示词、词库和页面配置后，把 HTML 应用转换为微信小程序版本。
- 增强静态部署说明，让非技术用户也能把自己的学习工具部署给家人使用。

- 每日一记卡片
